## Design Context

### Users
Call center managers and team leads monitoring real-time agent performance and call metrics on medium-sized TV displays mounted in the office. They glance at the dashboard throughout the day — it needs to be legible from a distance and deliver information at a glance without demanding attention.

### Brand Personality
Calm, clear, trustworthy.

### Aesthetic Direction
- **Visual tone**: Light and airy — generous whitespace, soft surfaces, gentle contrast. Think Apple Health: confident data presentation wrapped in a serene, approachable shell.
- **References**: Apple Health app — soft gradients, rounded cards, restrained color, clean typography, lots of breathing room.
- **Anti-references**: Dense enterprise dashboards, dark hacker aesthetics, neon/gaming UI, heavy drop shadows.
- **Theme**: Light mode only. Easy on the eyes — muted, desaturated palette with color reserved for status indicators and key metrics.

### Design Principles
1. **Breathe** — Generous spacing and whitespace. Every element needs room. If it feels cramped, it's wrong.
2. **Glanceable** — Information hierarchy must work from 3 meters away. Big numbers, clear labels, obvious status.
3. **Calm confidence** — The dashboard should feel reassuring, not alarming. Even when metrics are bad, the UI stays composed.
4. **Purposeful color** — Color is used sparingly and intentionally: status indicators, key metrics, and nothing else. The base palette stays neutral and soft.
5. **Invisible complexity** — Pagination, updates, and transitions should feel effortless. No jarring changes. Smooth and predictable.
